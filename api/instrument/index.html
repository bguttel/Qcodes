class qcodes.instrument.ChannelList(parent: InstrumentBase, name: str, chan_type: Type[InstrumentModuleType], chan_list: Optional[Sequence[InstrumentModuleType]] = None, snapshotable: bool = True, multichan_paramclass: Optional[Type[MultiChannelInstrumentParameter]] = None)[source]ÔÉÅ

Bases: ChannelTuple, MutableSequence[InstrumentModuleType]

Mutable Container for channelized parameters that allows for sweeps over all channels, as well as addressing of individual channels.

This behaves like a python list i.e. it implements the collections.abc.MutableSequence interface.

Note it may be useful to use the mutable ChannelList while constructing it. E.g. adding channels as they are created, but in most use cases it is recommended to convert this to a ChannelTuple before adding it to an instrument. This can be done using the to_channel_tuple() method.

Parameters

  • parent ‚Äì The instrument to which this ChannelList should be attached.

  • name ‚Äì The name of the ChannelList.

  • chan_type ‚Äì The type of channel contained within this list.

  • chan_list ‚Äì An optional iterable of channels of type chan_type. This will create a list and immediately lock the ChannelList.

  • snapshotable ‚Äì Optionally disables taking of snapshots for a given ChannelList. This is used when objects stored inside a ChannelList are accessible in multiple ways and should not be repeated in an instrument snapshot.

  • multichan_paramclass ‚Äì The class of the object to be returned by the __getattr__() method of ChannelList. Should be a subclass of MultiChannelInstrumentParameter. Defaults to MultiChannelInstrumentParameter if None.

Raises

  • ValueError ‚Äì If chan_type is not a subclass of InstrumentChannel

  • ValueError ‚Äì If multichan_paramclass is not a subclass of MultiChannelInstrumentParameter (note that a class is a subclass of itself).
